# Visual System

## Estado

- Tipo: `contractual`
- Fase inicial: `3`
- Estado: `done`
- Ultima actualizacion: `2026-03-12`

---

## Objetivo

Definir una direccion visual propia para el portfolio que deje atras el look del template y traduzca la estrategia del sitio a un lenguaje mas sobrio, tecnico y reconocible.

Este sistema debe reforzar:

- claridad operativa
- confiabilidad
- estructura
- evidencia por encima de ruido visual

No define:

- copy final
- contenido maestro
- motion premium detallado

---

## Direccion visual

La direccion aprobada para esta fase es:

- `tech-modern`
- limpia
- propia
- sobria
- sistematica

La referencia de tono visual debe sentirse:

- mas cercana a operations / product delivery que a gaming-neon
- suficientemente tecnica sin caer en una estetica SaaS generica
- premium por control y contraste, no por exceso de decoracion

---

## Paleta aprobada

### Light mode

- `bone`: `#f3eee4`
- `bone-strong`: `#e7dfd2`
- `graphite`: `#1f252d`
- `graphite-soft`: `#55606b`
- `mineral-teal`: `#2f7a77`
- `mineral-teal-soft`: `#d9ebe7`

### Dark mode

- `night`: `#11161b`
- `night-soft`: `#1a2129`
- `fog`: `#e8ece7`
- `fog-soft`: `#b3b9bd`
- `mineral-teal-dark`: `#6ca7a1`
- `mineral-teal-deep`: `#214f53`

### Reglas de uso

- `bone` y `night` son los fondos base principales.
- `graphite` y `fog` gobiernan jerarquia textual.
- `mineral-teal` es el acento principal para CTA, links, focus y highlights.
- no reintroducir el azul/cyan del template como color primario del sistema
- usar el acento con moderacion; no convertir cada bloque en superficie acentuada

---

## Tipografia aprobada

### Stack

- headings: `Manrope`
- cuerpo: `IBM Plex Sans`
- mono/eyebrows/metadatos: `IBM Plex Mono`

### Reglas de uso

- `Manrope` se usa en headings, wordmark y jerarquia principal.
- `IBM Plex Sans` se usa en body copy, navegacion, labels y texto corrido.
- `IBM Plex Mono` se usa para eyebrows, tags, metadatos y acentos informativos.
- evitar mezclar mas familias en esta fase

---

## Branding base

La identidad base aprobada para esta fase es:

- monograma tipografico: `P`
- wordmark corto: `Pawpaw`

### Reglas

- el branding visible debe ser tipografico
- no depende todavia de un logo ilustrado final
- el navbar y footer deben expresar la marca con un lockup simple y consistente
- el favicon puede ser provisional, pero ya no debe reutilizar el icono heredado del template

---

## Dark mode

- se mantiene como parte real del sistema
- debe tener paridad razonable con light mode
- no debe convertirse en una estetica separada o dramatica
- el toggle permanece visible en navbar

Principio:

- mismo sistema
- distinta atmosfera
- misma legibilidad

---

## Fondos y superficies

### Fondos

- usar fondos claros calidos y contenidos en light mode
- usar fondos oscuros densos y sobrios en dark mode
- preferir gradientes sutiles, tramas o overlays suaves sobre bloques ruidosos

### Superficies

- las cards deben sentirse limpias, estructuradas y con borde visible
- el hover debe ser sobrio: leve elevacion, borde o sombra, no transformaciones agresivas
- los paneles editoriales pueden usar tintes suaves del acento sin saturacion fuerte

---

## Componentes base

### Navbar

- fija, compacta y legible
- branding tipografico a la izquierda
- navegacion clara sin efecto template llamativo
- toggle visible pero de bajo ruido

### Hero

- layout editorial con texto dominante y asset visual contenido
- la imagen principal vive en un panel controlado, no como poster invasivo
- el CTA principal debe seguir siendo lo mas evidente del bloque

### Botones

- formas redondeadas pero no juguetonas
- peso visual suficiente para CTA principal
- outline mas tecnico y limpio para CTA secundaria o enlaces de superficie

### Cards y previews

- bordes claros
- radio consistente
- fondos diferenciados entre canvas y superficie
- tags y metadatos con uso de mono y tinte suave del acento

### Footer CTA

- debe cerrar con tono claro y profesional
- no compite visualmente con el hero
- funciona como superficie de conversion sobria

---

## Iconografia

- line icons de grosor consistente
- evitar mezclar iconos demasiado juguetones con iconos muy enterprise
- el acento de iconos debe venir de color y fondo, no de ilustracion recargada

---

## Aplicacion por pagina

### `/`

- concentra la mayor expresion del sistema visual
- debe sentirse como landing principal

### `/work`

- usa la misma identidad, con hero mas contenido y foco en profundidad de evidencia

### `/experience`

- usa la misma identidad, con atmosfera consistente y jerarquia clara para trayectoria

---

## Restricciones

- no reintroducir logos o iconos del template original en superficies visibles
- no usar visuales que contradigan la tesis `delivery / operations first`
- no compensar falta de contenido maestro con decoracion excesiva
- no convertir Fase 3 en una fase de motion premium o branding final exhaustivo
